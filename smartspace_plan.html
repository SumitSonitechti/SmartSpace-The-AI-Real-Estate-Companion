Takes free-text lifestyle preferences, encodes them as embedding vectors via Vertex AI, and matches against property + neighbourhood embeddings. Unique feature: simultaneous home + office pair optimisation with real commute matrix.

Lifestyle text → Vertex embed → Vector search → Ranked pairs

Multimodal: takes photos (Vertex Vision classifies rooms, detects quality issues, tags amenities) and voice notes (Cloud STT, Hindi + EN). Gemma synthesises a polished listing in <30 seconds. Owner reviews and publishes.

Ingests PDFs (sale deeds, agreements, NOCs). Document AI extracts entities and clauses. Fine-tuned Gemma classifies risk level per clause. Outputs a colour-coded summary with actionable flags — no legal advice, just structured information.

PDF upload → Document AI → Clause extract → Risk flag
Technical stack

What to build with

Google-first stack, open-source where it matters.

AI core

Gemma 2B / 7B

Local inference for chat, intent parsing, and listing copy. Deploy on Vertex AI Model Garden. Fine-tune on Indian real estate corpus for domain vocabulary.

Platform

Vertex AI

Embeddings (text-embedding-gecko), Vision API, Document AI, Model Registry, Pipelines for agent orchestration. Cloud Run for agent microservices.

Frontend

Next.js 14 + TypeScript

App Router. React Server Components for data-heavy pages. Framer Motion for antigravity UI. deck.gl for map layers. Tailwind CSS.

Maps

Google Maps Platform

Maps JavaScript API, Routes API for commute matrix, Places API for neighbourhood data, Geocoding. deck.gl HexagonLayer over Maps for heatmaps.

Data

Firestore + AlloyDB

Firestore for real-time listing updates and user sessions. AlloyDB for structured property analytics, time-series pricing, and ML-ready joins.

Search

Vertex AI Matching Engine

High-scale approximate nearest neighbour search for property + lifestyle embeddings. Sub-100ms latency at scale. Managed, no infra to run.

Voice

Cloud Speech-to-Text v2

Multilingual model with Hindi (hi-IN) and English (en-IN). Streaming recognition for voice search. Custom vocabulary for property terms (BHK, RERA, etc.).

Infra

Cloud Run + Pub/Sub

Containerised agent microservices on Cloud Run (auto-scale to zero). Pub/Sub for async agent chaining. Cloud Scheduler for nightly market data refresh.

Design language

Antigravity UI principles

The UI should feel like you're hovering above the city, looking down — light, spatial, floating.

Depth layers

Background: map at opacity 40%. Mid: blurred property cards. Foreground: chat panel. Three Z-planes give a sense of floating above the city grid.

Motion language

Cards rise on scroll (translateY -12px + opacity). Hover: 4px lift + soft shadow. Transitions: 280ms ease-out. Map layers fade in with 400ms stagger per tile.

Colour system

Earth + forest palette: deep green (#1a6b4a), warm sand (#f8f6f0), amber highlights for price signals, coral for risk flags. Never pure black or white.

Typography

DM Serif Display for headings (editorial weight). DM Sans for UI. Size scale: 48/32/20/16/13px. Tight tracking on display (-0.03em). Generous leading on body (1.75).

Glassmorphism

Floating panels: backdrop-filter blur(16px), background rgba(255,255,255,0.72), 0.5px border rgba(255,255,255,0.4). Only for panels over the map — not for general UI.

Accessible AI

All AI-generated text has a "Generated by AI" chip. Loading states use skeleton shimmer, not spinners. Hindi UI labels use Noto Sans Devanagari fallback stack.
